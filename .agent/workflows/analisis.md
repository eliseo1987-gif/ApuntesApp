---
description: Realizar un análisis profundo del estado del sistema y proponer mejoras
---

Este workflow guía al agente para realizar un análisis exhaustivo del ERP y sugerir optimizaciones.

1. **Recopilación de Datos**:
   - Leer `index.html` para entender la estructura actual.
   - Leer `app.js` para revisar la lógica de negocio y el estado del `CRM`.
   - Revisar los reportes actuales y el estado del pipeline.

2. **Identificación de Áreas de Mejora**:
   - Buscar cuellos de botella en la UX.
   - Identificar funciones que podrían automatizarse con IA.
   - Revisar la consistencia visual (Scientific Luxury / Premium).

3. **Presentación de Propuestas**:
   - El agente presentará un resumen ejecutivo de hallazgos.
   - Propondrá 3 mejoras específicas priorizadas por impacto.

4. **Ejecución (opcional)**:

- Si el usuario aprueba, proceder con la implementación siguiendo las skills correspondientes.
